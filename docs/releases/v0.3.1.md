# Agent Workbench v0.3.1

This patch fixes Qwen Code discovery and execution when Agent Workbench is started with Node installed through NVM.

## Fixes

- Automatically finds the `qwen` CLI installed beside the Node runtime that launched the Web server.
- Keeps the matching NVM Node directory first on Qwen's execution `PATH`, preventing an older system Node from running the Qwen shim.
- Discovers configured Qwen models even when the CLI health probe is temporarily unavailable.
- Replaces ambiguous `exit undefined` messages with explicit timeout, signal, missing executable, or no-exit-code diagnostics.
- Preserves environment overrides through `PENTESTERFLOW_QWEN_PATH` and `PENTESTERFLOW_QWEN_SETTINGS`.

On the validated development installation, Qwen Code 0.19.11 is detected successfully and all 19 configured unique model IDs are exposed in the model switcher.
