# Agent Workbench v0.4.0

This release expands the secure local workbench with bounded content discovery,
port scanning, validation evidence, session migration, and a more flexible
terminal UI while preserving the Agent Workbench CLI and Burp bridge.

## Highlights

- UI-agnostic shared Agent Runtime used by both CLI and Web, with CLI JSON compatibility tests.
- FFUF 2.1.0 with a pinned checksum, fixed built-in wordlist, and scope/rate/runtime limits.
- Nmap TCP connect scanning in the safe image.
- Separate, opt-in raw-socket Nmap image with `NET_RAW` only and no privileged container.
- Typed, canonical, single-use approvals for FFUF, Nmap, and bounded HTTP validation.
- Validation artifacts and Coverage updates without automatic finding confirmation.
- One-way, hash-tracked import of legacy CLI JSON sessions into SQLite.
- Redacted session export and exact-title-confirmed session/artifact deletion.
- Drag-resizable side panels, dense/comfort terminal modes, and scanner profile health.
- Qwen Code and OpenCode model discovery shown from their live configured model lists.

## Scanner images

```sh
npm run scanner:build:all
```

The raw image is disabled until the server is started explicitly with:

```sh
AGENT_WORKBENCH_ENABLE_RAW_SCANNER=1 npm run start:web
```

The safe profile remains non-root and capability-free. Neither profile receives
host mounts, the Docker socket, home directories, AI credentials, model-chosen
images, entrypoints, environment variables, or raw commands.

## Runtime requirements

- Node.js 20+ for the CLI.
- Node.js 22+ for the Web workbench.
- Docker for isolated scanners.
- One optional provider: Ollama, Qwen Code, Codex CLI, Claude CLI, OpenCode, or OpenClaude.
