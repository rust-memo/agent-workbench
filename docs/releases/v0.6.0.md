# Agent Workbench v0.6.0

v0.6.0 introduces durable, structured reconnaissance results while preserving
the existing CLI and Web Workbench flows.

## Highlights

- Per-tool lifecycle records for queued, running, saving, completed, failed,
  cancelled, and timed-out executions.
- Separate raw, parsed, and metadata artifacts for Scope, Subfinder, CRT.sh,
  DNSX, HTTPX, and approval-gated follow-up scanners.
- Normalized domain, subdomain, URL, and IP assets with scope state and
  multi-tool source attribution.
- Automatic combined domain files, duplicate accounting, DNS enrichment, and
  structured HTTPX JSONL/live-host exports.
- Partial stdout/stderr and valid results survive non-zero exits, cancellation,
  timeouts, output limits, and server restarts.
- A dedicated Recon Results workspace with Assets, Tool Runs, Combined, HTTPX,
  Files, and AI Review tabs.
- Authenticated, audited, paginated artifact viewing for text, JSON, JSONL,
  CSV, XML, and Markdown.
- Hash-bound, redacted AI review previews with explicit approval and auditable
  response artifacts.
- User and AI interest recommendations, notes, selected downloads, and
  approval-gated scan proposals.
- A server-owned trusted tool registry that enforces typed arguments,
  scope validation, fixed executables, output limits, and `shell: false`.

## Verification

- Unit and integration coverage for normalization, partial scanner output,
  source attribution, combined output, HTTPX persistence, trusted tool policy,
  AI redaction/approval, and scanner artifacts.
- Playwright coverage for the Recon Results workspace, combined artifact
  viewer, asset selection, and AI preview.
