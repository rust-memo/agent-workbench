# Agent Workbench v0.3.2

This v0.3 patch makes cloud-backed turns direct and observable while adding first-class Codex CLI and Claude CLI providers.

## Highlights

- Sends Qwen Code, Codex CLI, Claude CLI, OpenCode, and OpenClaude turns directly without a blocking confirmation dialog.
- Redacts credentials at the CLI-provider boundary before the exact payload is hashed, previewed, or sent.
- Publishes a non-blocking Cloud Preview with provider, model, payload size, redaction count, and SHA-256 audit identity.
- Adds Codex model discovery from the local config/cache and runs Codex in non-interactive, ephemeral, read-only mode.
- Adds Claude CLI discovery with an automatic OpenClaude-compatible fallback when the official binary is unavailable.
- Keeps model actions schema-validated and scanner action approvals separate, canonical, expiring, and single-use.
- Makes new sessions inherit the currently selected provider and model.
- Adds a dedicated operation progress dock and reduces the live terminal footprint.
- Preserves the existing CLI, Burp Integration, Docker ScannerRunner, DNSX, Katana, Nuclei, Findings, Coverage, cancellation, and SQLite event replay.

## Security boundary

Cloud previews contain only the redacted outbound envelope and are rendered as escaped React text. The unredacted prompt is not written to events or audit metadata. The audit trail stores the provider, model, size, redaction count, and SHA-256 identity. Scanner actions still require their independent risk approval; this release removes only the cloud-dispatch confirmation.
