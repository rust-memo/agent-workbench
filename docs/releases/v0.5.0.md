# Agent Workbench v0.5.0

v0.5.0 turns the local browser interface into a Recon-first security workspace while
preserving the PentesterFlow CLI, Browser MCP, and Burp Integration.

## Highlights

- Professional Recon workspace with an inline domain/scope form and Quick, Standard,
  and Advanced run profiles.
- Persisted SQLite recon runs with live Scope, Passive Discovery, DNS, HTTP, and
  Analysis stages, global event replay, cancellation, and restart recovery.
- Ranked assets and manual-test recommendations with review, dismiss, and completed
  states; the event terminal now occupies less than one-third of the workspace.
- Standard profiles propose bounded Katana and pinned safe Nuclei actions. Advanced
  profiles additionally propose bounded FFUF and unprivileged Nmap Connect actions.
  Every proposal still requires a single-use operator approval.
- Web skill catalog and session loading for the upstream PentesterFlow skills plus
  API Authorization, OAuth/OIDC, Business Logic, Request Smuggling, and File Upload.
  Curated entries include pinned source commit, license, risk, and provenance.
- Qwen model discovery reads the active model and every configured provider model,
  supports compatibility settings fields, uses a longer health timeout, and reports
  timeout/signal/unexpected-stop states instead of `exit undefined`.
- Provider, model, artifact, finding, coverage, approval, scanner-isolation, and
  loopback security boundaries from v0.4 remain intact.

## Runtime requirements

- Node.js 20+ for the CLI and Node.js 22+ for the Web workbench.
- Docker Engine for scanner actions.
- One optional reasoning provider: Ollama, Qwen Code, Codex CLI, Claude CLI,
  OpenCode, or OpenClaude.

Build scanner images with the v0.5 tags before starting the Web server:

```sh
npm run scanner:build:all
npm run build
npm run start:web
```

The server remains loopback-only on `http://127.0.0.1:9099` and prints a fresh,
single-use pairing URL on every start.
