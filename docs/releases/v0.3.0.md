# Agent Workbench v0.3.0

This release adds an isolated scanner pipeline to the local Web workbench while preserving the existing PentesterFlow CLI, Browser MCP, and Burp bridge.

## Highlights

- Ephemeral Docker ScannerRunner for Subfinder, DNSX, HTTPX, Katana, and Nuclei.
- Server-owned scanner image, commands, limits, and policy; targets are supplied through stdin.
- Read-only, non-root scanner containers with all Linux capabilities dropped, `no-new-privileges`, resource limits, and no host mounts, Docker socket, home directory, or provider credentials.
- Human approval workflow for Katana and Nuclei using expiring, single-use canonical action hashes bound to the current scope and mode.
- Findings triage with explicit validation evidence before a result can be marked confirmed.
- Persistent Coverage tracking for discovered assets and attempted tests.
- Cancellation for both AI turns and active scanner containers.
- Nuclei v3.11.0 with a fixed nuclei-templates commit and a restricted HTTP-only policy.

Scanner findings are never confirmed automatically. Scope validation is fail-closed for action inputs and best-effort at the network layer; hard egress enforcement still requires a controlled proxy or network policy.

## Upgrade

```sh
npm install
npm run build
npm run scanner:build
npm run start:web
```

The Web workbench requires Node.js 22+ and Docker Engine. The CLI remains compatible with Node.js 20+.
